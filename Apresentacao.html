Sistema de Gestão de Projetos Ágeis

Implementação com Arquitetura MVC

Projeto Integrador VI-A
Anderson de Lima Carvalho
Universidade Católica de Pelotas

Objetivos do Projeto

  • Implementar sistema SaaS de gerenciamento ágil com arquitetura MVC
  • Aplicar conceitos de Clean Architecture e boas práticas de engenharia de software
  • Desenvolver funcionalidades multi-tenancy e controle de acesso
  • Criar sistema de gestão de sprints, épicos e user stories
  • Implementar métricas e relatórios para análise de produtividade
Foco: Demonstrar aplicação prática de conceitos teóricos em um sistema real seguindo metodologias ágeis

Arquitetura Model-View-Controller

VIEW (React + TypeScript)
↕
CONTROLLER (Hono Framework)
↕
MODEL (Drizzle ORM + PostgreSQL)
  • Model: Drizzle ORM mapeando tabelas PostgreSQL com type safety
  • View: Componentes React funcionais com hooks modernos
  • Controller: Classes que processam requisições HTTP via Hono

Estrutura em Camadas do Backend

Controllers

Recebem requisições HTTP e coordenam operações

Services

Contêm lógica de negócio e regras do domínio

Repositories

Encapsulam acesso a dados com padrão Repository

Models

Definem estrutura do banco via Drizzle ORM

Seguindo princípios de Clean Architecture de Robert C. Martin

Stack Tecnológica

Backend

  • Node.js + TypeScript
  • Hono Framework
  • Drizzle ORM
  • PostgreSQL

Frontend

  • React + TypeScript
  • Vite (bundler)
  • Tailwind CSS
  • React Router
Pacote @shared: Implementa Shared Kernel para type safety end-to-end

Implementação de Metodologias Ágeis

  • Scrum Framework: Baseado no Scrum Guide de Schwaber & Sutherland
  • Sprints: Ciclos iterativos com metas específicas
  • Product Backlog: Gerenciamento de user stories priorizadas
  • Sprint Backlog: Itens selecionados para cada sprint
  • Kanban Board: Visualização do fluxo de trabalho
Sistema flexível que apoia processos ágeis sem engessá-los

Boas Práticas de Engenharia de Software

  • Clean Code: Nomes significativos, funções pequenas e específicas
  • Type Safety: TypeScript end-to-end eliminando erros de runtime
  • Validação: Múltiplas camadas seguindo "never trust user input"
  • Injeção de Dependência: Baixo acoplamento e facilidade de testes
  • Transações: Garantia de consistência em operações críticas
  • Logging: Rastreabilidade e debug estruturado
Documentação consistente mantida como parte integrante do desenvolvimento

Resultados e Aprendizados

Sucessos

  • Arquitetura MVC eficaz para sistemas complexos
  • Type safety eliminou classe inteira de erros
  • Pacote @shared evitou dessincronização
  • Separação clara de responsabilidades

Desafios

  • Aprendizado do Drizzle ORM
  • Implementação de multi-tenancy
  • Tempo escasso para todas as funcionalidades
  • Testes limitados por restrições de tempo
Projeto validou a importância de arquitetura bem fundamentada desde o início

Conclusões

  • Validação prática de conceitos teóricos de engenharia de software
  • Arquitetura MVC adaptada eficazmente para sistemas modernos
  • Importância do design inicial e estabelecimento de boas práticas
  • TypeScript essencial para projetos de médio/grande porte
  • Metodologias ágeis como meio para entregar valor, não fim em si
Contribuição: Sistema que implementa metodologias ágeis sendo exemplo de boas práticas de engenharia de software
